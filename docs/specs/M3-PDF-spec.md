# ユズリ PDF書き出し仕様 v1.0（M3）

> 目的：家族が読んで使える「遺された一冊」を生成する。アプリが将来使えなくなっても価値が残る、紙としての完成度を最優先する。
> 技術：PDFKit / UIGraphicsPDFRenderer。外部アセット不使用。テキストは**画像化せず選択可能**にする。

## 1. 出力モード

| モード | 内容 | 既定 |
|---|---|---|
| 安全版 | 秘匿項目を伏せる（「（秘匿・非表示）」と表記、値は出さない） | ◯（既定） |
| 全部入り版 | 秘匿項目も含める | 生体認証の再要求が必要 |

- 空欄項目は既定で**省略**（設定で「未記入も表示」に切替可）。
- どちらのモードでも、文末に免責（`disclaimer.export`）を印字。

## 2. ドキュメント構成（順序）

1. **表紙**：タイトル（ロケール名）／本人氏名／作成日・最終更新日／「これは遺言書ではありません」の一文。
2. **はじめに（免責）**：`disclaimer.onboarding` ＋ `disclaimer.export` を印字。
3. **目次**：カテゴリと開始ページ。
4. **本文**：カテゴリを `order` 順。各カテゴリ見出し＋項目（ラベル：値）。該当カテゴリに `disclaimerKey` があれば末尾に注記。
5. **緊急医療カード**：1ページに収まる携帯用サマリ（氏名・血液型・アレルギー・常用薬・かかりつけ医・緊急連絡先）。切り取り線を表示。
6. **書類のありか一覧**：物理保管場所マップを最終ページにまとめる。

## 3. レイアウト / タイポグラフィ

| 項目 | 指針 |
|---|---|
| 用紙 | ロケール連動：ja＝A4、en-US＝Letter、その他＝A4既定 |
| 余白 | 上下20mm／左右18mm目安 |
| 本文フォント | システムフォント、本文11–12pt、見出し16–20pt（高齢の読み手を想定し小さくしすぎない） |
| 行間 | 1.4目安 |
| コントラスト | 黒文字／白背景。装飾色は最小限 |
| ページ番号 | 各ページ下部中央。「— n / N —」 |
| ヘッダ | 本人氏名＋最終更新日（表紙以外） |

## 4. ローカライズ

- ラベル・見出し・注記は表示ロケールで解決（labelKey→String Catalog、フォールバックdefaultLabel）。
- 日付・数値はロケール書式（`DateFormatter`/`NumberFormatter`）。
- 免責はロケール別キー（免責文ドラフト参照）。
- 項目セットは解決済みテンプレ（base＋overlay）に従う（ja＝72項目、en＝68項目）。

## 5. 秘匿・安全設計

- 安全版：秘匿フィールドは値を出さず「（秘匿・非表示）」。
- 全部入り版：書き出し前に生体認証再要求。生成後は共有シートに渡すのみ（自動保存・送信はしない）。
- フッターに「機微情報を含む場合があります。保管・共有にご注意ください」（`disclaimer.export`）。

## 6. 技術メモ

- `UIGraphicsPDFRenderer` でページ単位に描画。長い項目は自動改ページ（テキスト計測→残余高で分割）。
- テキストは `NSAttributedString` 描画で**選択・検索可能**に保つ（画像化しない）。
- フォント埋め込みはシステムフォント既定で可。外部フォントは持たない。
- 生成は端末内のみ。ネットワーク不使用。
- 大規模（全カテゴリ・全項目）でも数秒以内を目標。

## 7. 受け入れ基準（M3）

1. 安全版で秘匿が一切出力されない（テスト：秘匿3項目に値を入れて出力→値が現れない）。
2. 全部入り版は再認証を要し、認証後に秘匿が出力される。
3. 表紙・免責・目次・本文・緊急医療カード・ありか一覧が指定順で生成される。
4. ja＝A4／en-US＝Letter で生成される。
5. テキストが選択可能（画像化されていない）。
6. 空欄が既定で省略され、設定で表示に切替できる。
7. 全文がローカライズされ、未登録キーは `defaultLabel` にフォールバックする。

## 8. スコープ外（後続）

写真・QR添付、テンプレ別デザインテーマ、パスワード保護PDF（※暗号化アーカイブ側で担保）、印刷専用の冊子割付。
